1. 오늘의 목표
 - 프로젝트 초기 세팅
 - H2 환경 구축 및 초기 엔티티 검증
 - 한국투자증권 오픈 API 발급

2. 진행사항 & 결과
 - 프로젝트 생성 완료
 - 추후 PostgreSQL로 이전을 고려해서 H2 DB 설정에 MODE=PostgreSQL 옵션 적용
 - 주식 종목을 저장할 Stock 엔티티(종목코드, 종목명, 시장타입) 설계 및 JPA 연동
 - /h2-console 접속을 통해 애플리케이션 구동시 STOCK 테이블이 생성되는걸 확인
 - 한국투자증권 오픈 API 발급 완료

3. 트러블 슈팅 및 고민할 점
 - 초기부터 PostgreSQL 사용을 고려했었으나, 초기 인프라에 시간을 쏟는 것 보다 비즈니스 로직 구현 속도를 높이기 위해 H2를 사용하기로 결정
 - 대신 이후 이전이 용이하도록 H2에 MODE=PostgreSQL를 적용

4. 내일 할 일
 - 한국투자증권 오픈 API 테스트
 - 외부 API에서 데이터를 받아오기 위한 Spring Boot 내 HTTP 통신 기본 환경 조사 및 세팅

5. 일지
 - 첫 백엔드 프로젝트의 시작이다. 평소 관심이 있었던 주식관련 프로젝트를 만들기로 했다. 기본적인 기능에 더해서 백트래킹 및 데이터 시각화에 집중할 생각이다.
 - 완성하면 개인적으로도 사용할 예정이고, 웹사이트도 운영을 해보면서 지속적으로 개선할 생각이다. 내가 여러 주식 앱 웹을 써보면서 불편했던 점들과 있었으면 좋겠다 싶은 것들을 추가해볼 생각이다.
 - 일단 MVP를 최대한 빨리 만들어보자.
