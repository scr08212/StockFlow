1. 오늘의 목표
 - 한국투자증권 오픈 API와 스프링 부트 서버 간의 REST API 통신 통로 개척
 - 보안 설정 분리 및 비동기 통신 도구(WebClient) 인프라 구축
 - 삼성전자(005930) 주식 현재가 시세 단건 조회 기능 구현 및 검증

2. 진행사항 & 결과
 - API Key 보안 관리 기틀 마련: 깃허브 노출 리스크를 방지하기 위해 AppKey와 SecretKey를 환경변수로 저장.
 - 통신 도구 선정 (WebClient): 대용량 주가 시계열 데이터를 효율적으로 가져오기 위해 기존의 동기식 RestTemplate 대신 스프링 부트 3에서 권장하는 비동기/논블로킹 기반의 WebClient를 설정 클래스(WebClientConfig)를 통해 빈(Bean)으로 등록함.
 - 1차 목표(접근 토큰 발급) 성공: 한투 API 서버 요구 규격에 맞게 snake_case를 자바의 camelCase으로 매핑해 주는 @JsonProperty 기반 DTO를 설계하여 24시간 임시 신분증인 Access Token을 정상적으로 발급받음.
 - 2차 목표(삼성전자 현재가 조회) 성공: 발급받은 토큰과 한투 고정 거래 ID(tr_id)를 HTTP 헤더에 실어 GET 요청을 날려, 삼성전자의 현재가, 전일대비 금액, 등락률 데이터를 성공적으로 파싱하고 콘솔 출력을 통해 검증함.

3. 트러블 슈팅 및 고민할 점
 - WebClient 사용 이유
    주식 데이터 수집 특성상 나중에 코스피 948개 종목의 주가를 긁어와야 함.
    동기(Blocking) 방식을 쓰면 앞선 종목의 응답이 올 때까지 스레드가 멈춰 대기하므로 성능 병목이 발생함. 따라서 인프라 구축 단계부터 비동기(Non-blocking) 확장이 유연한 WebClient를 선택함.

 - tr_id 헤더와 내부 정적 클래스(PriceDetail)의 필요성:
    이유 1: 금융권 API 특성상 게이트웨이가 요청을 올바른 서버 장비로 포워딩할 수 있도록 고유 거래 코드인 tr_id를 헤더에 필수로 명시해야 통신이 가능했음.
    이유 2: 한투 응답 Json(주식현재가 시세[v1_국내주식-008])의 경우 응답 Body에 output이라는 중괄호({}) 뎁스를 하나 더 가지고 있었음. 따라서 Jackson 라이브러리가 한번에 파싱할 수 있도록 Dto 내부에 스태틱 클래스를 선언하여 구조를 일치시킴.

 - 현재 한투 토큰을 발급받은 후 이를 따로 저장하는 로직이 없음. 다른 종목코드로 변경 후 다시 실행해보니 오류 발생. 토큰을 따로 어딘가에 저장하는식으로 추후 해결해야함.
